# Contributing to Memcachify 🚀

First off, thank you for considering contributing to Memcachify! It's people like you that make the open-source community such an amazing place to learn, inspire, and create.

## 🛠️ Development Setup

1. **Fork** the repository and clone it locally.
2. Install dependencies:
   ```bash
   npm install
   ```
3. Run tests to ensure everything is working:
   ```bash
   npm test
   ```
4. Create a branch for your feature or bug fix:
   ```bash
   git checkout -b feature/my-new-feature
   ```

## 🧪 Guidelines

- **Zero-Dependency**: We strive to keep Memcachify free of external runtime dependencies. Please avoid adding new packages to `dependencies`.
- **Tests**: Every new feature or bug fix should include tests.
- **Types**: Ensure your changes are fully typed.
- **Code Style**: Follow the existing code style. We use ESLint to maintain consistency.

## 🤝 How to contribute

1. **Issues**: If you find a bug, please [open an issue](https://github.com/darshan1005/memcachify/issues).
2. **Pull Requests**:
   - Ensure your PR description clearly explains the change.
   - Link any related issues.
   - Run `npm run build` and `npm test` before submitting.

## 📜 Code of Conduct

Help us keep Memcachify open and inclusive. Please be respectful and constructive in your communications.

Happy coding!
